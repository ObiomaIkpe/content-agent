# To-Do

## i. Manual content input via Telegram
Add a `/write` command (or similar) that lets you tell the AI what you want to post about directly from Telegram, bypassing the snapshot pipeline entirely. For example:
> `/write I want to post about why I switched from REST to GraphQL on my last client project`

The bot should take that prompt, run it through the strategist and writers as if it were a highlight from the work report, and deliver drafts for review as normal.

## ii. LinkedIn token expiry check
LinkedIn access tokens expire every 60 days. Currently there is no warning — posts just silently fail when the token goes stale.

Add a check on bot startup (and optionally on each `/run`) that hits the LinkedIn API with the current token and alerts you on Telegram if it's expired or close to expiring, with instructions on how to refresh it.

## iii. Improved image generation
The current images generated by Pollinations.ai have several problems observed from real output:

- **Text is garbled** — any text rendered in the image comes out as unreadable scrambled characters. Pollinations cannot render legible text at all. Text should be removed from image prompts entirely.
- **Too generic** — every image is a variation of "developer sitting at a desk with monitors", regardless of what the post is actually about. There is no real connection to the post content.
- **Looks like stock art** — flat orange-and-blue illustration style. Nothing distinctive or on-brand.
- **No platform differentiation in practice** — despite per-platform style prompts being defined in `image_generator.py`, the output feels largely the same across platforms.

Possible directions:
- **Pillow-based code cards** — skip AI image generation entirely for text-heavy posts. Use Pillow to generate a clean quote card: post excerpt on a solid or gradient background with your name/handle. This is what high-performing dev Twitter accounts actually use, and it would look far more polished than the current output.
- **Better AI provider** — if sticking with AI-generated images, explore a provider with better prompt adherence (e.g. Stable Diffusion via Replicate, which has a free tier).
- **Hybrid** — use Pillow cards for Twitter/LinkedIn (text-first platforms) and AI images for Instagram/Facebook (visual-first platforms).

## iv. Aggregator noise filtering
The ActivityWatch collector captures everything — including YouTube music videos, movie searches, Netflix browsing, random web searches, and other personal activity that has nothing to do with work. This pollutes the snapshot and can confuse the aggregator agent into thinking non-work activity is worth posting about.

Add a filtering layer in `local_agent/activitywatch_collector.py` (and optionally `github_collector.py`) that strips out known noise before the snapshot is saved. Examples of what to filter:

- Browser URLs matching `youtube.com/watch`, `netflix.com`, `music.apple.com`, general Google searches for non-technical terms
- Window titles matching media players, Spotify, video apps
- Any app/site with under a minimum time threshold (e.g. < 30 seconds — probably just switching tabs)

The filter rules should be configurable — ideally a blocklist in `config/` so they can be updated without touching code.
